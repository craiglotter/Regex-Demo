Welcome to Regular-Expressions.info
The Premier Web Site about Regular Expressions

A regular expression, or regex for short, is a special text string for describing a search pattern. You can think of regular expressions as wildcards on steroids. You are probably familiar with wildcards notations such as *.txt to find all text files in a file manager. The regex equivalent is .*\.txt.

But you can do much more with regular expressions. In a text editor like EditPad Pro or a specialized regex tool like PowerGREP, you could use a regular expression like \b[A-Z0-9._%-]+@[A-Z0-9._%-]+\.[A-Z]{2,4}\b to find an email address. A very similar regex (replace the first \b with ^ and the last one with $) can be used in a Perl or PHP script, or a Java or .NET application to check in just one line of code whether the user entered a properly formatted email address.

Regular Expression Tutorial

Do not worry if the above made very little sense to you. Any non-trivial regex looks daunting to anybody not familiar with them. But with just a bit of experience, you will soon be able to craft your own regular expressions like you have never done anything else. The free Regular-Expressions.info Tutorial explains everything bit by bit.

This tutorial is quite unique, because it not only explains the regex syntax, but also describes in detail how the regex engine actually goes about its work. You will learn quite a bit, even if you have already been using regular expressions for some time. This will help you understand quickly why a particular regex does not do what you initially expected. It will save you lots of guesswork and head scratching when you need to write more complex regexes.

Tools & Languages That Support Regexes

There are many software tools and programming languages that support regular expressions these days. It is really impossible to list them all on this web site. Obviously, Perl deserves to get a full discussion of its regex capabilities. This programming language, originally designed for text-processing only, is the main cause for the popularity that regular expressions enjoy nowadays. The main reason is that Perl's regex engine introduced many new powerful features. Furthermore, regexes are part of the Perl syntax itself, rather than add-on library as with most other languages.

Other popular development frameworks, like Java and .NET (dot net) ship with classes that allow you to instantly use regular expressions. Learning regular expressions and using these classes will save you many hours doing similar text processing in your own code without them.

Not Only for Programmers

Even if you are not a programmer, you can still use regular expressions in many situations. They will make finding information a lot easier, and you can use them in powerful search and replace operations to quickly make changes across large numbers of files. A simple example is gr[ae]y which will find both spellings of the word grey in one operation, instead of two. There are many text editors and search and replace tools with decent regex support. Two popular and useful tools are the text editor EditPad Pro and the all-round regex utility PowerGREP.
